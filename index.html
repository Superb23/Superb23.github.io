搭建Blog

基础工具

建议安装在非C盘的同一个文件夹下面

nodejs下载: https://www.cnblogs.com/liuqiyun/p/8133904.html（内含有nodejs和npm）
git下载:建议直接在官网下载最新的
——安装教程:https://www.cnblogs.com/xueweisuoyong/p/11914045.html

nodejs的环境测试失败的—–“使用管理员权限打开cmd”

检测是否安装成功:
node -v
npm -v
git –version

（建议下载sublime，该软件不是必须工具，纯个人喜好，用于编辑文本方便）

——————————–以上都是安装基础的工具——————————-

安装blog主题hexo

hexo下载: 使用管理员权限打开cmd，再输入:npm install hexo-cli -g
出现版本号说明安装成功（例如:hexo-cli@4.3.0）
检查是否安装成功:hexo -v

搭建仓库

在官网github.com上面注册账号，user name一定要记住！！！这个将会你blog的域名（名字别太丑）

登录github账号,创建一个repository;其中repository name格式必须是:名字.github.io
勾选Add a README file,再点击Create repository

此时，你已经创建完成了你的仓库

本地与github连接

打开任意一个文件，右键点击git bash here,输入: ssh
再输入: ssh-keygen -t rsa -C “邮箱地址” 这里要点四次回车，不要关闭黑框；

此时将会生成一个.ssh的文件在C盘，根据四次回车后黑框里面返回的信息，可以找到.ssh文件的地址
(一般是在C:\user\administrator.ssh)
在github网站里面的setting–>SSh and GPG key–>SSH keys（创建一个SSH key，名字随便取）
打开.ssh文件里面的id_rsa.pub,这里面存放的是本地访问的密钥，复制粘贴到刚创的SSH key里面;

在黑框输入:ssh -T git@github.com 回车后，再输入yes，再回车；

此时你已经完成了github仓库和本地的连接；

本地blog

1.创建一个文件夹专门放本地blog内容（非C盘），记住这个文件夹的地址；
2.找到第一步安装的git，管理员权限打开git-bash.exe,输入:cd 存放blog文件夹的地址;
(目的是进入刚创的空文件夹，例如:cd E:\blog)
3.输入: hexo init;
(目的是初始化hexo，此处容易因网络超时失败，多次输入直至成功，出现Start blogging with hexo!)
4.输入:hexo s;
(生产本地的hexo主题的页面，只是本地！！此时还无法通过域名访问，需要本地上传到仓库才行)
如果你要关闭你的blog，则可以输入hexo s,然后再按ctrl c,就能关闭本地的blog

本地上传互联网

1.修改blog文件下面的_config.yml里面的布置
找到最下面的deploy,需要修改两个，branch需要自己写进去，记得:后面有一个空格
deploy:
type: git
repository: (打开github网站，点我们创建的那个repository–>绿色按键Code–>复制HTTPS在此处)
branch: main

2.找到第一步安装的git，管理员权限打开git-bash.exe,输入:cd 存放blog文件夹的地址;
(例如:cd E:\blog)
输入: npm install hexo-deployer-git –save
这里是在安装hexo的上传工具

3.输入: hexo g 生成文件
输入: hexo d 上传文件(此处易因网络超市失败上传，多输入几次就行，会跳出登录界面
正常登录即可)
上传可能存在延迟，只要上传没显示失败，过一会就能在github的仓库里面看到我们上传的文件

此处blog搭建就已经全部完成了，可以通过自己创建的域名访问我们的blog了(例如:https://superb23.github.io)

写blog

1.建一个.md的文件用于存放你要写的blog内容；
在blog文件下打开git bash,输入: hexo n 写的blog名字
(例如: hexo n 搭建Blog)

2.输入: hexo s
生成本地静态网页，在编辑的时候可以通过刷新本地的静态网页来看写出来的效果
(记得ctrl s,保存再刷新网页)
